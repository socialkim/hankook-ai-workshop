Q1. 회사 내부 자료를 AI에 넣어도 되나요? 보안이 걱정됩니다.
────────────────────────────────────────────────────

A: 3가지 원칙을 지키시면 됩니다.

  원칙 1 — 공개 정보만 입력
  이미 보도자료나 웹사이트에 나간 정보는 입력해도 됩니다.
  아직 공개되지 않은 내부 전략, 재무 수치, 인사 정보는 넣지 마세요.

  원칙 2 — 민감 정보는 마스킹
  꼭 넣어야 하는 내부 정보가 있다면 핵심 수치나 이름을 [X]로 바꾸세요.
  예: "매출 1,200억" → "매출 [금액]", "박OO 부장" → "[담당자]"
  AI는 구조와 맥락만 있으면 충분히 좋은 결과를 줍니다.

  원칙 3 — 기업용 플랜 사용
  ChatGPT Team/Enterprise, Claude Team/Enterprise, Gemini Business는
  사용자의 대화 내용을 모델 학습에 사용하지 않습니다.
  회사 차원에서 기업용 플랜을 도입하면 보안 걱정이 크게 줄어듭니다.

  현실적으로 — 이미 수많은 기업 직원들이 AI에 업무 자료를 넣고 있습니다.
  "안 쓰자"가 아니라 "안전하게 쓰자"가 올바른 접근입니다.


Q2. ChatGPT가 우리 대화 내용을 학습에 사용하나요?
────────────────────────────────────────────────────

A: 무료/개인 유료(Plus) 플랜은 기본적으로 학습에 사용될 수 있습니다.
  하지만 끄는 방법이 있습니다.

  ChatGPT: Settings → Data Controls → "Improve the model for everyone" → OFF
  Claude: Settings → Privacy → "Use conversations to improve Claude" → OFF
  Gemini: Google AI 설정에서 "Gemini Apps Activity" → OFF

  기업용 플랜(Team/Enterprise)은 애초에 학습에 사용하지 않으므로
  별도 설정이 필요 없습니다.


Q3. 이력서나 개인정보가 포함된 문서를 AI에 넣으면 법적 문제가 생기나요?
────────────────────────────────────────────────────

A: 네, 주의가 필요합니다.

  개인정보보호법에 따라 이름, 주민번호, 연락처 등 개인식별정보를
  외부 AI 서비스에 입력하는 것은 "개인정보의 제3자 제공"에 해당할 수 있습니다.

  안전하게 사용하는 방법:
  1. 이름 → "A지원자", "B지원자"로 치환
  2. 학교명 → "수도권 4년제 대학"으로 일반화
  3. 연락처, 주민번호, 주소 → 완전 삭제
  4. 핵심 경력과 역량 정보만 남기기

  이렇게 마스킹하면 개인을 특정할 수 없으므로 법적 리스크가 거의 없고,
  분석 품질도 충분히 유지됩니다.


Q4. 기사 원고를 AI에 넣으면 저작권 문제가 있나요?
────────────────────────────────────────────────────

A: 자사 기사를 본인이 업무용으로 AI에 입력하는 것 자체는
  현재 기준으로 법적 문제가 되지 않습니다.

  다만 주의할 점:
  - 타사 기사를 무단으로 대량 입력하여 파생 콘텐츠를 만드는 것은 주의
  - AI가 생성한 결과물의 저작권은 현재 법적으로 불명확한 영역
  - 결론: 참고·보조 도구로 활용하되, 최종 결과물은 사람이 검토·수정하는 것이 안전

Q5. ChatGPT, Claude, Gemini 중 어떤 걸 쓰는 게 좋나요?
────────────────────────────────────────────────────

A: "어떤 게 최고"가 아니라 "어떤 상황에 어떤 도구"가 맞는지가 중요합니다.
  볼펜, 만년필, 연필이 각각 용도가 다른 것처럼요.

  ChatGPT (Plus $20/월)
  - 범용 문서 작성, 아이디어 발상, 이미지 생성(DALL-E)
  - GPTs로 반복 작업 자동화 가능
  - 플러그인 생태계가 가장 풍부

  Claude (Pro $20/월)
  - 긴 문서 분석, 정확한 요약, 코드 작성
  - 20만 토큰(약 A4 300장)까지 한 번에 처리
  - 논리적 분석과 글쓰기에 특히 강함

  Gemini (Advanced $20/월)
  - 구글 검색 연동, Deep Research(딥리서치) 기능
  - 실시간 정보 검색이 필요한 리서치에 강함
  - 이미지 생성 기능 내장 (Imagen)

  추천: 3개 모두 유료 플랜을 갖추고, 상황에 따라 골라 쓰는 것이 최선입니다.
  하나만 고르라면? ChatGPT가 가장 범용적입니다.


Q6. Perplexity는 뭔가요? 써야 하나요?
────────────────────────────────────────────────────

A: Perplexity는 "AI 검색 엔진"입니다.

  일반 AI(ChatGPT 등)가 "학습된 지식으로 답변"하는 반면,
  Perplexity는 "실시간으로 인터넷을 검색해서 답변 + 출처 링크 제공"합니다.

  언제 쓰면 좋은가:
  - 최신 뉴스, 통계, 수치를 찾을 때
  - "출처가 있는 답변"이 필요할 때
  - 팩트체크를 할 때

  유료(Pro $20/월)와 무료 모두 사용 가능합니다.
  기자분들에게는 특히 유용한 도구입니다.


Q7. 무료로도 충분한가요, 꼭 유료를 써야 하나요?
────────────────────────────────────────────────────

A: 솔직히 말씀드리면, 무료와 유료의 차이가 상당히 큽니다.

  무료 버전의 한계:
  - 사용량 제한 (하루 몇 회)
  - 최신 모델이 아닌 이전 모델 사용
  - 고급 기능(딥리서치, 이미지 생성, GPTs 등) 제한
  - 긴 문서 처리 불가

  업무용으로 쓰려면 유료 플랜을 강력 추천합니다.
  월 $20(약 2.8만원)인데, 이것으로 절약되는 시간을 생각하면 투자 대비 효과가 큽니다.
  커피 한 잔을 아끼면 되는 수준인데, 매일 1-2시간을 절약할 수 있습니다.

Q8. AI가 틀린 정보를 줄 때가 있다던데, 어떻게 확인하나요?
────────────────────────────────────────────────────

A: 맞습니다. AI는 "할루시네이션(환각)"이라고 하는 현상이 있어서,
  그럴듯하지만 사실이 아닌 정보를 만들어내기도 합니다.

  확인 방법:
  1. 출처 요청: "위 답변의 출처를 알려줘"라고 추가 질문
  2. 교차 검증: 같은 질문을 2~3개 AI에 각각 던져서 비교
  3. Perplexity 활용: 출처 링크가 자동으로 제공되므로 클릭해서 확인
  4. 팩트체크 프롬프트: "위 답변에서 확인이 필요한 수치나 사실을 표시해줘"

  기자분들은 이미 팩트체크에 익숙하시니, 그 습관을 AI에도 적용하면 됩니다.
  "AI의 답변은 1차 초안이지 최종본이 아니다"라고 생각하세요.


Q9. AI가 만든 글에 편향이 있을 수 있나요?
────────────────────────────────────────────────────

A: 네, AI 모델에는 학습 데이터에 기반한 편향이 존재합니다.

  주의할 편향:
  - 영어권 시각 편향: 한국 관련 정보에서 서구 시각이 반영될 수 있음
  - 최신성 편향: 학습 데이터 마감일 이후 변화 미반영
  - "양쪽 다 맞다" 편향: 논쟁적 주제에서 지나치게 중립적인 답변

  대응 방법:
  - "한국 상황에 맞게 답변해줘"라고 명시
  - "반대 관점도 제시해줘"라고 추가 요청
  - 여러 AI의 답변을 비교하여 공통점과 차이점 파악
  - 최종적으로 사람(기자)의 판단이 반드시 필요


Q10. 수치나 통계를 AI에게 물어봐도 되나요?
────────────────────────────────────────────────────

A: 물어볼 수는 있지만, 반드시 원본 출처를 확인해야 합니다.

  AI가 수치를 잘하는 경우:
  - 잘 알려진 공개 통계 (GDP, 인구, 환율 등)
  - 최신 검색이 가능한 도구 사용 시 (Gemini, Perplexity)

  AI가 수치를 틀리는 경우:
  - 세부적인 업계 통계 (특정 회사 매출, 시장 점유율 세부)
  - 최신 데이터 (학습 이후 변경된 수치)
  - 한국 시장 고유 데이터

  실전 팁: AI에게 "이 수치의 출처를 알려줘. 확인이 안 되면 [미확인]으로 표시해줘"
  라고 요청하면 AI 스스로 불확실한 부분을 표시해줍니다.

Q11. AI로 기사를 쓰면 안 되는 거 아닌가요?
────────────────────────────────────────────────────

A: "AI가 기사를 쓰는 것"과 "기자가 AI를 도구로 활용하는 것"은 다릅니다.

  ✕ AI가 기사를 쓰는 것 — 문제 있음
  AI가 자동으로 기사를 생성해서 그대로 발행하는 것은
  정확성, 윤리, 저작권 등 여러 문제가 있습니다.

  ○ 기자가 AI를 도구로 활용하는 것 — 권장
  - 취재 전 배경 조사
  - 인터뷰 질문 리스트 준비
  - 긴 문서 요약/분석
  - 소제목/헤드라인 아이디어 발상
  - 번역/영문 교정
  - 데이터 분석/시각화

  핵심: AI는 "보조 도구"이고, 최종 판단과 책임은 기자에게 있습니다.
  포토샵으로 사진을 편집한다고 사진기자의 역할이 사라지지 않듯,
  AI로 글쓰기를 보조한다고 기자의 역할이 사라지지 않습니다.


Q12. 프롬프트를 잘 못 쓰겠어요. 쉽게 시작하는 방법이 있나요?
────────────────────────────────────────────────────

A: 가장 쉬운 방법 3가지를 드립니다.

  방법 1 — ABA 사용하기
  "나는 [직책]인데, [이런 일]을 하고 싶어. 바로 답하지 말고 먼저 질문해줘."
  이것만 쓰면 AI가 알아서 필요한 걸 물어봅니다. 가장 쉬운 시작법입니다.

  방법 2 — 메타프롬프팅
  "이 일을 시키려는데, 네가 잘하려면 내가 뭘 알려줘야 해?"
  AI가 질문 리스트를 만들어주면 거기에 답변만 하면 됩니다.

  방법 3 — 대화로 접근
  처음부터 완벽한 프롬프트를 쓰려고 하지 마세요.
  대충 시작해서 → 결과를 보고 → "이 부분은 이렇게 바꿔줘"라고 수정 요청.
  대화를 3-4번 주고받으면 원하는 결과가 나옵니다.

  "AI는 써본 만큼 보인다" — 처음부터 잘하려고 하지 마시고,
  일단 많이 써보세요. 쓰면 쓸수록 감이 잡힙니다.


Q13. 영어로 프롬프트를 써야 더 좋은 결과가 나오나요?
────────────────────────────────────────────────────

A: 2024년까지는 그런 경향이 있었지만, 2026년 현재는 한국어로도 충분합니다.

  다만 예외:
  - 영문 결과물이 필요한 경우: 영어로 지시하면 번역 없이 바로 나옴
  - 매우 전문적인 기술 용어: 영어가 약간 더 정확할 수 있음
  - 코딩 관련: 영어가 조금 더 유리

  결론: 한국어로 쓰세요. 편하게 쓰는 것이 더 중요합니다.
  프롬프트의 품질은 언어가 아니라 "구체성"이 결정합니다.


Q14. AI를 업무에 쓰고 있다고 밝혀야 하나요?
────────────────────────────────────────────────────

A: 현재 한국에는 AI 사용 공시 의무가 법적으로 없습니다.
  다만 업계 윤리 차원에서:

  기사 작성: AI를 "작성"에 사용한 경우 밝히는 것이 언론 윤리에 부합
  기사 보조: 자료 조사, 요약 등 보조 도구로 사용한 경우 별도 공시 불필요
  내부 업무: 공지문, 메일, 제안서 등 내부 문서에는 공시 불필요
  대외 공식 문서: 회사 정책에 따라 판단

  한국일보 자체 가이드라인이 있다면 그것을 따르시면 되고,
  없다면 이번 교육을 계기로 가이드라인 수립을 건의하시는 것도 좋겠습니다.

Q15. AI 때문에 기자 일자리가 줄어들까요?
────────────────────────────────────────────────────

A: 솔직하게 말씀드리겠습니다.

  줄어드는 일자리: 단순 번역, 단순 요약, 정형화된 보도(주가, 날씨, 스포츠 결과)
  늘어나는 일자리: 탐사보도, 심층 분석, 데이터저널리즘, AI 활용 콘텐츠 기획

  역사를 보면 답이 있습니다.
  인쇄기가 발명됐을 때 필사 수도사의 일은 줄었지만, 기자라는 직업이 새로 생겼습니다.
  카메라가 발명됐을 때 초상화 화가의 일은 줄었지만, 사진기자가 생겼습니다.

  핵심은: "AI를 잘 쓰는 기자"가 "AI를 안 쓰는 기자"의 일을 대체하는 것이지,
  AI가 기자를 대체하는 것은 아닙니다.

  오늘 이 자리에 계신 것 자체가 이미 올바른 방향으로 가고 계신 겁니다.


Q16. 교육 후에도 계속 배울 수 있는 방법이 있나요?
────────────────────────────────────────────────────

A: 가장 좋은 학습법은 "매일 업무에 한 가지씩 적용해보는 것"입니다.

  추천 루틴:
  1. 매일 아침: AI에게 "오늘 할 일 정리해줘" (일상 습관화)
  2. 주 1회: 새로운 프롬프트 기법 하나를 업무에 시도
  3. 월 1회: 다른 AI 도구를 써보기 (ChatGPT만 쓰시면 Claude도 시도)

  유용한 리소스:
  - OpenAI 공식 가이드: platform.openai.com/docs/guides/prompt-engineering
  - Anthropic 프롬프트 가이드: docs.anthropic.com/en/docs/build-with-claude/prompt-engineering
  - 한국어 AI 뉴스레터: AI타임스, 테크42 등 구독
  - 이번 교육 자료: 공유 폴더에 계속 업데이트 예정

  그리고 — "AI는 써본 만큼 보인다" 이 말 기억하시죠?
  가장 좋은 선생님은 직접 해보는 경험입니다.
